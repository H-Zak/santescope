## Interaction Contracts

### Search autocomplete (UI-01, D-06..D-09)

- Input: full-screen centered on landing (`/`)
- Debounce: 150ms before filtering (D-08)
- Algorithm: `startsWith` match priority, then `includes` fallback — no Fuse.js
- Each result row: `{nom} ({dept_code}) [{score_badge}]`
- Score badge colored per DPE scale
- On select: navigate to `/commune/[code]`
- Loading state: Skeleton search bar while `index.json` fetches
- Cache: `localStorage` key `santescope_index` after first load

### Double panel (UI-02..05, D-10..D-13)

- Layout: `flex flex-row` 50/50, `gap-8`, `min-h-screen`
- Mobile (<768px): `flex-col` — diagnostic panel on top, twin panel below (D-27)
- Left panel heading: "Ma commune" (locked label)
- Right panel heading: "Commune jumelle" (locked label)
- Swap twin: click any card in the horizontal twin row below — replaces right panel without page navigation
- Loading: Skeleton cards fill both panels while commune JSON fetches

### Score display (D-11)

- DPE badge: square, `border-radius: 8px`, background = DPE class color
- Gauge bar: horizontal, width proportional to score/10, filled with DPE color
- Numeric: `{score}/10` in 20px semibold text
- Classification label below: "Tres vulnerable" (E), "Vulnerable" (D), "Modere" (C), "Correct" (B), "Exemplaire" (A)

### Score detail (D-12)

- 4 components: APL (acces aux soins), Pauvrete, 75+ isoles, Urgences
- Each row: label + value + comparison to national median (arrow icon + delta text)
- Deficit: red delta text; surplus: green delta text

### Domino alert (D-14)

- Container: `rounded-lg border-l-4 p-4`
- Normal: `bg-orange-50 border-orange-400`
- Critical (>50%): `bg-red-50 border-red-400`
- Content line 1: "N/M medecins ont 55+ ans (X% — moy dept: Y%)"
- Content line 2: "Estimation 2030 : -N medecins"

### Missing specialties (D-15, DOC-03)

- Bullet list, each item: `{Specialty} — {Pathology}: {multiplier}x moy. nat.`
- Max 5 items shown; "+N autres" link if more

### Twins row (D-13)

- Horizontal scroll row, 3 cards visible
- Each card: commune name + similarity % + DPE badge
- "+N" button at end to expand
- Active twin: card border `2px solid #0F766E`

### Data quality warning (D-16)

- Yellow banner at top of results page when `data_quality === "partial"` or `"minimal"`
- Text: "Données partielles — certains indicateurs sont estimés ou indisponibles"
- Missing sections: grayed out with text "Non disponible"
- Score null: no DPE badge, text "Données insuffisantes" in slate-400

### Free comparison mode (D-17..D-18)

- Entry: "Comparer avec..." button on results page (bottom of right panel)
- Opens second search input inline (no modal)
- URL updates to `/comparer/[code1]/[code2]`
- Right panel replaces twin with selected commune
- Header persists search bar for re-search (D-25)

### Map (D-19)

- Static PNG served from `public/data/communes/{code}-map.png`
- Container: `w-full h-[100px] rounded-lg object-cover`
- Fallback: placeholder with teal background + commune initial if PNG missing

### PDF export (D-20..D-23)

- Trigger: "Telecharger le rapport" button — secondary button style
- Hidden DOM component: `#pdf-export-root` — styled with dark theme (`bg-#232323`)
- Capture: `html2canvas({ scale: 2, useCORS: true })`
- Output: PNG download, filename `santescope-{code}-{YYYY-MM-DD}.png`
- PDF structure (D-21): teal header → DPE badge + A-E strip → Resume executif (3 lines) → Indicateurs cles (3-col grid) → Scores par dimension (4 horizontal bars) → Points forts / Alertes (2-col) → Footer sources
- Adapts mockup 12 dimensions → our 4 components (APL, Pauvrete, 75+ isoles, Urgences)
